# 牧民智能政策助手 - 技术方案

## 项目定位

面向内蒙古农牧民的**蒙汉双语AI政策问答小程序**。
用户只需语音提问，AI基于真实政策文件作答，支持蒙古语和汉语。

> 核心价值：不是通用翻译工具，是**懂当地政策的AI助手**。
> 和直接问ChatGPT的区别：我们的AI只基于**真实官方政策文件**回答，不瞎编。

---

## 一、功能清单

### 核心功能
1. **语音问答** - 说蒙语或汉语提问，AI语音回答（蒙/汉）
2. **政策知识库** - 导入真实政策文件，AI只基于这些文件回答（RAG）
3. **政策图文卡片** - 大字+图片+蒙汉对照，展示办事流程
4. **一键拨号** - 直拨村委会、乡镇服务大厅、社保局
5. **政策提醒** - 补贴申报截止日期、新政策推送

### 政策覆盖范围
- 各类补贴（草原生态补贴、粮食补贴、农机补贴等）
- 医保社保（新农合、养老保险、大病救助）
- 低保/五保/特困供养
- 教育补助（学生资助、助学贷款）
- 草原保护政策
- 创业贷款/小额信贷
- 其他惠民政策

## 二、关于你的两个关键问题

### 问题1："不和官方对接，问AI和问ChatGPT有什么区别？"

**你说得对，区别就是：数据来源。**

| | 直接问ChatGPT | 我们的小程序 |
|---|---|---|
| 数据来源 | 互联网泛知识，可能过时/编造 | **本地真实政策文件** |
| 回答准确性 | 经常瞎编政策细节 | **只基于导入的文件回答** |
| 本地化 | 不知道当地具体标准 | **针对具体旗县的政策** |
| 找不到答案时 | 硬编一个答案 | **明确说"未找到相关政策，建议咨询XX电话"** |

**实现方式：RAG（检索增强生成）**

```
用户提问: "低保每个月多少钱？"
        │
        ▼
[第1步] 从政策知识库中搜索相关文档片段
        → 命中：《XX旗2024年最低生活保障实施办法》
        → "城镇低保标准为每人每月780元，农村低保标准为每人每年6500元..."
        │
        ▼
[第2步] 把文档片段 + 用户问题 一起发给AI
        → Prompt: "根据以下政策文件回答用户问题，不要编造，如果文件中没有就说不确定"
        │
        ▼
[第3步] AI基于真实文件生成回答
        → "根据《XX旗最低生活保障办法》，农村低保标准为每人每年6500元，
           需要携带身份证、户口本到嘎查村委会申请。"
```

**政策数据哪来？**
- 教育系同学负责：去政府网站、政务公开页面收集政策文件（PDF/Word）
- CS同学负责：写脚本把这些文件切片、建索引、存到向量数据库
- 这不需要"官方对接"，政策文件本身就是公开的
- 后期如果能对接政务数据就更好，但比赛阶段用公开文件完全够

## 三、蒙古语支持 - 最大技术挑战

### 现实情况

蒙古语在AI领域属于**低资源语言**，支持很差：

| 能力 | 汉语 | 英语 | 蒙古语 |
|------|------|------|--------|
| 语音识别(STT) | 成熟 | 成熟 | **几乎没有好的方案** |
| AI翻译 | 成熟 | 成熟 | **质量一般** |
| 语音合成(TTS) | 成熟 | 成熟 | **方案极少** |

### 蒙古语可行方案

| 方案 | 可行性 | 说明 |
|------|--------|------|
| **讯飞开放平台** | ⭐⭐⭐⭐ | 支持蒙古语语音识别和合成，有免费额度，**最推荐** |
| 百度语音 | ⭐⭐ | 蒙语支持有限 |
| 微软Azure语音 | ⭐⭐⭐ | 支持蒙古语TTS，有免费额度 |
| 自训练模型 | ⭐ | 不现实，数据和时间都不够 |

> **讯飞是目前蒙古语语音最好的选择**。讯飞和内蒙古有合作，
> 蒙古语语音识别和合成都有，开发者免费额度够比赛用。

### 蒙汉翻译方案

AI大模型对蒙古语翻译质量一般，但有个取巧的办法：

```
方案A（简单直接）: 蒙古语 → AI直接翻译 → 汉语
  优点：简单
  缺点：翻译质量不稳定

方案B（推荐）: 政策内容预先准备蒙汉双语版本
  即：政策知识库里每条政策都存蒙语版和汉语版
  用户说蒙语 → 识别为蒙语文字 → 搜索蒙语知识库 → AI用蒙语回答
  用户说汉语 → 识别为汉语文字 → 搜索汉语知识库 → AI用汉语回答
  优点：回答质量高，不依赖翻译
  缺点：需要教育系同学准备双语政策文档

方案C（折中）: B为主 + A兜底
  有双语版本的用B，没有的用A做翻译
```

**方案B最适合比赛**——教育系同学本来就负责内容，让他们准备蒙汉双语版本的政策文档，
这样根本不需要AI翻译蒙古语，直接从蒙语知识库中检索回答。

## 七、核心技术实现

### RAG问答流程（后端核心）

```python
# 1. 政策文档导入（一次性）
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.vectorstores import Chroma
from langchain.embeddings import HuggingFaceEmbeddings

# 用免费的中文embedding模型
embeddings = HuggingFaceEmbeddings(model_name="shibing624/text2vec-base-chinese")

# 把政策文件切片并存入向量数据库
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
docs = splitter.split_documents(policy_documents)
vectordb = Chroma.from_documents(docs, embeddings, persist_directory="./policy_db")

# 2. 用户提问时检索+生成
def answer_policy_question(question: str, lang: str = "zh"):
    # 从向量库中检索最相关的政策片段
    relevant_docs = vectordb.similarity_search(question, k=3)
    context = "\n".join([doc.page_content for doc in relevant_docs])

    # 让AI基于政策文件回答
    prompt = f"""你是农牧民政策咨询助手。请严格根据以下政策文件内容回答问题。
如果政策文件中没有相关信息，请明确说"暂未找到相关政策，建议拨打当地政务服务热线12345咨询"。
不要编造任何政策内容。用简单易懂的口语回答，像跟农牧民面对面说话一样。

【政策文件】
{context}

【用户问题】
{question}

请用{'蒙古语' if lang == 'mn' else '简单汉语'}回答："""

    # 调用智谱GLM（免费）
    response = zhipu_client.chat.completions.create(
        model="glm-4-flash",
        messages=[{"role": "user", "content": prompt}],
        stream=True
    )
    return response
```

## 九、比赛亮点分析

### 为什么这个项目能拿第一

1. **解决真实问题** - 农牧民确实面临政策信息获取难的问题
2. **技术有深度** - RAG + 蒙古语NLP，不是简单调API
3. **有社会价值** - 评委喜欢有"情怀"的项目
4. **可演示** - 现场扫码，蒙语提问，立刻得到政策回答
5. **跨学科合作** - 教育系+CS，体现合作价值

### 演示脚本建议

```

"一位蒙古族牧民想知道草原生态补贴怎么申请，
 他不太会用手机搜索，打12345又听不太懂汉语..."

第二步：打开小程序，用蒙语说

```

## 十、风险与诚实评估

| 风险 | 严重程度 | 应对 |
|------|---------|------|
| 蒙语语音识别效果差 | ⚠️ 高 | 讯飞蒙语是目前最好的，如果不行就先做汉语为主+蒙语文字显示 |
| 政策信息过时/错误 | ⚠️ 高 | 每条回答标注来源文件和日期，提醒"以最新政策为准" |
| AI回答不准确 | ⚠️ 中 | RAG限定范围 + "找不到就说不知道" + 引导拨打电话 |
| 蒙语AI生成质量 | ⚠️ 中 | 用方案B（预存双语内容），不依赖AI实时翻译 |
| 服务器挂了 | ⚠️ 低 | 比赛前充分测试，录屏备份 |
