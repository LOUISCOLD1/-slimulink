# 开发流程与设计思路

## 一、平台选择：做什么形态？

### 方案对比

| 形态 | 开发周期 | 演示效果 | 难度 | 适合比赛？ |
|------|---------|---------|------|-----------|
| **微信小程序** | 1.5-2周 | ⭐⭐⭐⭐⭐ | ⭐⭐ | **最推荐** |
| Web App (H5) | 1-1.5周 | ⭐⭐⭐⭐ | ⭐ | 推荐 |
| Flutter跨平台App | 3-4周 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 时间紧张 |
| 原生iOS/Android | 4-6周 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 不现实 |
| UniApp小程序 | 1.5-2周 | ⭐⭐⭐⭐ | ⭐⭐ | 可选 |

### 推荐：微信小程序

**理由：**
1. **评委体验好** - 扫码就能用，不需要安装任何东西
2. **有"产品感"** - 比打开网页更像一个正式产品
3. **录音API成熟** - 微信提供原生录音、语音播放能力
4. **开发快** - 用微信开发者工具，2周足够
5. **国内比赛加分** - 评委都用微信，对小程序天然有好感

**如果时间更紧**，退而求其次做 Web App (H5)，手机浏览器打开即用。

## 二、整体设计思路

### 产品定位

> **"会思考的翻译官"** —— 不只是翻译你说了什么，而是理解你想说什么

### 核心用户场景

```
场景1: 出国旅游 —— 对着手机说中文，播放英文给外国人听
场景2: 面对面交流 —— 两个人各说各的语言，手机实时双向翻译
场景3: 学习辅助 —— 看到AI怎么"净化"口语，学习更地道的表达
```

### 页面规划（4个页面，精简）

```
┌─────────────────────────────────┐
│          首页/翻译页             │  ← 核心页面，80%时间在这
├─────────────────────────────────┤
│          对话模式页              │  ← 双人面对面翻译
├─────────────────────────────────┤
│          历史记录页              │  ← 翻译记录回顾
├─────────────────────────────────┤
│          设置页                  │  ← 语言切换、音色选择
└─────────────────────────────────┘
```

## 四、技术架构设计

### 微信小程序架构

```
微信小程序前端
├── pages/
│   ├── index/          # 首页-翻译页
│   ├── conversation/   # 对话模式
│   ├── history/        # 历史记录
│   └── settings/       # 设置
├── services/
│   ├── recorder.js     # 录音服务（wx.getRecorderManager）
│   ├── stt.js          # 语音识别（调用后端）
│   ├── translate.js    # AI翻译（调用后端）
│   └── tts.js          # 语音合成（调用后端）
├── utils/
│   └── api.js          # HTTP请求封装
└── app.js

Python后端 (FastAPI)
├── app/
│   ├── main.py              # 入口
│   ├── routers/
│   │   ├── translate.py     # 翻译接口
│   │   ├── stt.py           # 语音识别接口
│   │   └── tts.py           # 语音合成接口
│   ├── services/
│   │   ├── llm_service.py   # AI翻译（智谱/DeepSeek）
│   │   ├── stt_service.py   # 语音转文字
│   │   └── tts_service.py   # 文字转语音（Edge TTS）
│   ├── core/
│   │   ├── config.py        # 配置管理
│   │   └── prompt.py        # Prompt模板
│   └── models/
│       └── schemas.py       # 数据模型
├── requirements.txt
└── Dockerfile
```

### 数据流

```
[用户按住说话]
      │
      ▼
[微信录音API] ──录音文件(silk/mp3)──→ [后端 /api/stt]
                                          │
                                    [语音识别服务]
                                    (讯飞/百度/Whisper)
                                          │
                                     原始文本
                                          │
                                          ▼
                                    [后端 /api/translate]
                                          │
                                    [AI翻译服务]
                                    (智谱GLM/DeepSeek)
                                          │
                                   ┌──────┴──────┐
                                   │             │
                              净化文本        翻译结果
                                   │             │
                                   ▼             ▼
                              [返回前端展示三栏结果]
                                                 │
                                                 ▼
                                    [后端 /api/tts]
                                          │
                                    [Edge TTS]
                                          │
                                     音频文件
                                          │
                                          ▼
                                   [小程序播放语音]
```

## 六、语音识别方案（STT）

微信小程序里录音拿到的是音频文件，需要转文字。方案：

| 方案 | 免费额度 | 质量 | 推荐 |
|------|---------|------|------|
| **讯飞语音识别** | 每日500次免费 | ⭐⭐⭐⭐⭐ | **首选** |
| 百度语音识别 | 每日无限（短语音） | ⭐⭐⭐⭐ | 备选 |
| 腾讯语音识别 | 每月10000次免费 | ⭐⭐⭐⭐ | 备选 |
| 微信同声传译插件 | 免费 | ⭐⭐⭐⭐ | 最简方案 |

> **最简方案**: 微信小程序有官方的「同声传译」插件（plugin），
> 直接集成就有语音识别能力，免费，几行代码搞定。
> 但功能有限，如果要更好效果就用讯飞。

## 八、比赛答辩建议

2. **我们的方案** - AI驱动的智能语音翻译
3. **技术架构** - SOA架构图，展示专业性
4. **核心亮点** - 口语净化、上下文理解、三栏对比
5. **现场Demo** - 扫码打开小程序，现场翻译
6. **对比演示** - 同一句话：传统API vs 我们的AI翻译
7. **技术细节** - Prompt工程、流式输出、多引擎切换
8. **未来展望** - 支持更多语言、离线模式、行业定制
