# 开发流程与设计思路

## 一、产品设计

### 产品名（建议）
- "牧智通" / "政策通" / "蒙助手"
- 英文：MongolAssist / PolicyMate

### 目标用户画像
- 内蒙古农牧民，40-70岁
- 文化程度偏低，可能不识汉字
- 会用微信，但不太会打字搜索
- 蒙古语为母语，汉语为第二语言
- 需求：了解补贴政策、办事流程、联系方式

### 设计原则
1. **大字、大按钮** - 适合中老年用户
2. **能用语音就不用文字** - 张嘴就能问
3. **蒙汉对照** - 所有文字都双语显示
4. **简单直接** - 不要花哨功能，每个页面只做一件事

## 三、技术开发流程

### 后端API设计

```
POST /api/stt              # 语音转文字（上传音频文件）
POST /api/ask              # 政策问答（文字输入，返回AI回答）
POST /api/tts              # 文字转语音（返回音频文件）
GET  /api/policies         # 获取政策卡片列表
GET  /api/policies/:id     # 获取单条政策详情
GET  /api/contacts         # 获取联系电话列表
GET  /api/reminders        # 获取政策提醒
POST /api/ask/voice        # 一站式：上传语音→返回文字回答+语音回答
```
